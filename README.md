# AI-based Phishing URL Detection with UI

This project was developed for the **Bilişim Teknolojileri Uygulaması** course at **Konya Teknik Üniversitesi**.

The goal of the project is to build a machine learning-based system that detects whether a URL is **Phishing** or **Legitimate**, and provides a simple user interface for both single URL prediction and batch CSV prediction.

---

## Project Title

**AI-based Phishing URL Detection with UI**

Turkish title:

**Yapay Zekâ Tabanlı Kimlik Avı (Phishing) URL Tespiti ve Web Arayüzü Geliştirilmesi**

---

## Project Objective

The system is designed to:

- classify a raw URL as **Phishing** or **Legitimate**,
- show prediction probabilities,
- support single URL analysis,
- support batch CSV upload,
- provide a simple UI for demonstration and final project presentation.

---

## Label Meaning

The dataset labels are interpreted as:

| Label | Meaning    |
| ----- | ---------- |
| 0     | Phishing   |
| 1     | Legitimate |

This mapping is used consistently in the model, inference pipeline, and UI.

Important Modeling Decision

Some dataset features are not directly extractable from a raw URL in a real user interface.

For example, features related to page title or page content may not be available when the user only enters a URL.

To avoid training/inference mismatch, the final model was trained using features generated by the same module used during deployment:

src/model/feature_extractor.py

This means the final workflow is:

Raw URL → Feature Extraction → Model Prediction → UI Output
Final Model

The final deployment-ready model is:

HistGradientBoostingClassifier

It is trained on features extracted directly from raw URLs.

Final model artifacts:

Single URL Prediction

The system supports direct raw URL prediction.

Example:

result = predict_from_url("https://www.google.com")
print(result)

Output includes:

prediction
prediction_label
phishing_probability
legitimate_probability
Batch URL Prediction

The project supports batch prediction from a CSV file containing a column named:

Example command:

Limitations

This system uses structural URL features only.

It does not use:

live blacklist lookup,
WHOIS information,
DNS reputation,
website content analysis,
browser behavior analysis.

Therefore, the model should be considered a machine learning-based URL risk classifier, not a complete real-time security product.
